Section 2 of 5: Data Models

Core database schema — all 6 phases, so nothing needs to be bolted on later

Contact
Core relationship record
id uuid PK
name
type personal|work|family|volunteer
emails json[]
phones json[]
healthScore int 0-100
lastInteraction date?
interactionFreqDays int?
isFamilyMember bool
tags string[]
notes text
preferences json
createdAt / updatedAt
Interaction
Every conversation/event with a contact
id uuid PK
contactId FK → Contact
type call|meeting|email|message|event
direction inbound|outbound
summary
outcome text?
followUpRequired bool
followUpDate date?
followUpCompleted bool
callbackExpected bool
createdByHolly bool
location text?
duration int? (mins)
occurredAt
ActionItem
Short-term tasks from interactions
id uuid PK
interactionId FK? → Interaction
taskId FK? → Task
title
status todo|done|cancelled
priority low|medium|high|critical
assignedTo ian|holly
dueDate date?
createdAt
Project
Work and personal initiatives
id uuid PK
title
description
category personal|work|volunteer
status planning|active|on-hold|done|cancelled
priority low|medium|high|critical
targetDate date?
notes
createdAt / updatedAt
Task
Project subtasks and milestones
id uuid PK
projectId FK → Project
title
description
status todo|in-progress|done|cancelled
priority low|medium|high|critical
assignedTo ian|holly
dueDate date?
isMilestone bool
createdAt
KnowledgeItem
Insights queued for Obsidian export
id uuid PK
sourceId FK (polymorphic)
sourceType interaction|project|contact
content markdown text
proposedCategory
tags string[]
obsidianReady bool
exported bool
createdAt
Relationships: Contact 1-to-many Interaction. Interaction 1-to-many ActionItem. Project 1-to-many Task. ActionItem optionally links to Task (when an action grows into a project task). KnowledgeItem polymorphically references any source entity.
Does this schema cover everything? Note there is no separate User table since this is a single-user app - auth is handled by NextAuth.js with no need to store multiple user records. Holly is identified by API key, not a user row.
