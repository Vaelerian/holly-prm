Desktop: sidebar nav. Mobile: bottom tab bar. Central + button is the quick log shortcut.

Key Screens

Dashboard
Relationship health alerts, pending follow-ups, active project status, today's Holly briefing. Read-only overview.
Contacts
Searchable list. Click into a contact for full profile + interaction history. Quick-add form in under 30 seconds.
Log Interaction (modal/sheet)
Accessible from anywhere via the + button. Type, outcome, follow-up flag, action items. Supports offline entry. Target: completable in under 30 seconds.
Projects
Card grid by category (work / personal / volunteer). Click through to task list. Status badge prominent.
Tasks
Flat prioritised list across all projects. Filter by assigned-to (ian / holly), due date, priority. Mobile-friendly check-off.
Knowledge Queue
Items Holly or the system have flagged for Obsidian. Review, edit, mark ready, export as markdown. (Phase 6)
Settings
Relationship health thresholds, Holly API key management, notification preferences, integration config.
Does this screen structure work for you? The dashboard and Log Interaction are the two highest-priority screens to get right — they'll be used dozens of times a day.
