Holly PRM - Project Scope

This is a large system. Before we design any one piece, let's confirm how to break it into phases.

Based on your docs, Holly has 6 independent subsystems that map naturally to build phases. Each phase is shippable on its own and feeds into the next.

1

Foundation

Auth system, database schema, API scaffolding, PWA shell, deployment pipeline to vaelerian.uk. Everything else builds on this.

2

PRM Core

Contacts, interaction logging, call logs, relationship health monitoring. This is the heart of Holly.

3

PPM Core

Projects, tasks, milestones, action item extraction from interactions. Linked to PRM data.

4

Holly API Layer

RESTful endpoints + WebSocket for Holly (Openclaw) integration. Call logging, briefings, action items via API.

5

Calendar & Email Integration

Gmail + Google Calendar OAuth. Read-only calendar, email context, scheduling suggestions.

6

Knowledge Synthesis

Obsidian bridge, insight extraction from interactions, markdown export. Second brain capability.
